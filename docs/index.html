Why MCPSpec?

Deterministic

Unlike LLM-based testing, MCPSpec runs are fast and repeatable. No flaky tests, no token costs. Ideal for CI pipelines.

Secure

Catch Tool Poisoning (prompt injection in descriptions) and Excessive Agency (destructive tools without safeguards) before they reach production.

Collaborative

Record server interactions and share mocks with your team. Frontend developers and CI can test against mocks — no API keys required.

Record & Mock

Record a session once. Replay it to catch regressions. Mock it for CI. No API keys required.

The Workflow

# 1. Record a session against your real server
mcpspec record start "npx my-server"
# > .call get_user {"id": "1"}
# > .call list_items {}
# > .save my-api

# 2. Replay against a new version — catch regressions instantly
mcpspec record replay my-api "npx my-server-v2"

# 3. Start a mock server — drop-in replacement
mcpspec mock my-api

# 4. Generate a standalone .js file — commit to your repo
mcpspec mock my-api --generate ./mocks/server.js
node ./mocks/server.js

Mock server options

mcpspec mock my-api --mode sequential       # Tape-style matching
mcpspec mock my-api --latency original      # Simulate original timing
mcpspec mock my-api --latency 100           # Fixed 100ms delay
mcpspec mock my-api --on-missing empty      # Return empty for unrecorded tools
mcpspec mock my-api --generate ./mock.js    # Standalone file

Match mode (default): exact input match first, then next queued response per tool. Sequential mode: tape/cassette style. Generated files only require @modelcontextprotocol/sdk.

CI/CD Integration

Run mcpspec ci-init to generate a pipeline, or use these examples directly.

GitHub Actions

Generate this with mcpspec ci-init --platform github

name: MCP Server Tests
on: [push, pull_request]

jobs:
  mcpspec:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
      - run: npm install -g mcpspec

      - name: Run tests
        run: mcpspec test --ci --reporter junit --output results.xml

      - name: Security audit
        run: mcpspec audit "npx my-server" --fail-on high

      - name: Quality gate
        run: mcpspec score "npx my-server" --min-score 80

      - uses: mikepenz/action-junit-report@v4
        if: always()
        with:
          report_paths: results.xml

ci-init options

mcpspec ci-init                          # Interactive wizard
mcpspec ci-init --platform github        # GitHub Actions
mcpspec ci-init --platform gitlab        # GitLab CI
mcpspec ci-init --platform shell         # Shell script
mcpspec ci-init --checks test,audit,score
mcpspec ci-init --fail-on medium         # Severity gate
mcpspec ci-init --min-score 70           # Score threshold
mcpspec ci-init --force                  # Overwrite existing

Auto-detects platform from .github/ or .gitlab-ci.yml. GitLab --force surgically replaces only the mcpspec job.

Prevent Tool Poisoning

8 security rules including LLM-specific threats. Catch prompt injection in tool descriptions and destructive tools without safeguards.

mcpspec audit "npx my-server"                       # Passive (safe for production)
mcpspec audit "npx my-server" --mode active          # Active probing
mcpspec audit "npx my-server" --fail-on medium       # CI gate
mcpspec audit "npx my-server" --dry-run              # Preview targets
RuleModeDetects
Tool PoisoningpassiveLLM prompt injection in descriptions, hidden Unicode, cross-tool manipulation
Excessive AgencypassiveDestructive tools without confirmation params, arbitrary code execution
Path Traversalpassive../../etc/passwd style directory escape attacks
Input ValidationpassiveMissing constraints (enum, pattern, min/max) on inputs
Info DisclosurepassiveLeaked paths, stack traces, API keys in descriptions
Resource ExhaustionactiveUnbounded loops, large allocations
Auth BypassactiveMissing auth checks, hardcoded credentials
InjectionactiveSQL and command injection in inputs

Passive mode (default): 5 rules, analyzes metadata only, safe for production. Active/aggressive: all 8 rules, sends test payloads, auto-skips destructive tools. Each finding includes severity, evidence, and remediation advice.

Everything else you need

Test collections, quality scoring, interactive inspection, benchmarks, documentation, and a full web dashboard.

Test Collections

YAML-based suites with 10 assertion types, environments, variable extraction, tags, retries, and parallel execution. 5 reporter formats.

MCP Score

0-100 quality rating across 5 categories: docs, schema quality, error handling, responsiveness, and security. Opinionated schema linting. SVG badges.

Interactive Inspector

Connect to any server and explore tools, resources, and schemas in a live REPL. Call tools with JSON, view formatted responses.

Performance Benchmarks

Min/max/mean/median/P95/P99 latency, standard deviation, and throughput across hundreds of iterations with configurable warmup.

Doc Generator

Auto-generate Markdown or HTML documentation from server introspection. Lists all tools with input schemas and all resources with URIs.

Web Dashboard

Full React UI with 10 pages: servers, collections, runs, inspector, recordings, audit, benchmarks, score, docs. Dark mode included.
