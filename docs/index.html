Commands

13 commands for every stage of MCP server development.

mcpspec test [collection]

Run test collections with environments, tags, parallel execution, and multiple reporter formats.

mcpspec test                              # Uses ./mcpspec.yaml
mcpspec test ./tests.yaml                 # Specific file
mcpspec test --env staging                # Use staging variables
mcpspec test --tag @smoke                 # Filter by tag
mcpspec test --parallel 4                 # Parallel execution
mcpspec test --reporter junit --output results.xml
mcpspec test --reporter html --output report.html
mcpspec test --baseline main              # Compare against baseline
mcpspec test --watch                      # Re-run on changes
mcpspec test --ci                         # CI mode (no colors)

Reporters: console, json, junit, html, tap. Tags auto-normalize: @smoke matches smoke. Watch mode uses 300ms debounce.

mcpspec audit <server>

Security scanner with 8 vulnerability categories including LLM-specific threats. 3 scan modes with built-in safety controls.

mcpspec audit "npx my-server"                       # Passive (safe)
mcpspec audit "npx my-server" --mode active          # Active probing
mcpspec audit "npx my-server" --mode aggressive      # Aggressive
mcpspec audit "npx my-server" --fail-on medium       # CI gate
mcpspec audit "npx my-server" --acknowledge-risk     # Skip confirmation
mcpspec audit "npx my-server" --exclude-tools delete_user  # Skip tools
mcpspec audit "npx my-server" --dry-run              # Preview targets
RuleModeDetects
Path Traversalpassive../../etc/passwd style directory escape attacks
Input ValidationpassiveMissing constraints (enum, pattern, min/max) on tool inputs
Info DisclosurepassiveLeaked paths, stack traces, API keys in descriptions
Tool PoisoningpassiveLLM prompt injection, hidden Unicode, cross-tool manipulation in descriptions
Excessive AgencypassiveDestructive tools without confirmation params, arbitrary code execution
Resource ExhaustionactiveUnbounded loops, large allocations, recursion
Auth BypassactiveMissing auth checks, hardcoded credentials
InjectionactiveSQL and command injection in tool inputs

Active/aggressive modes auto-skip destructive tools (delete_*, destroy_*, etc.) and require confirmation. Each finding includes severity, evidence, and remediation advice.

mcpspec record start / replay / list / delete

Record inspector sessions, replay against servers, and diff results step by step for regression detection.

# Record a session
mcpspec record start "npx my-server"
# In REPL: .call get_user {"id": "1"}, .call list_items {}, .save my-session

# Replay against a new version and diff
mcpspec record replay my-session "npx my-server-v2"

# Manage recordings
mcpspec record list
mcpspec record delete my-session
Replaying 3 steps against my-server-v2...

  2/3 list_items........... [CHANGED] 38ms
  3/3 create_item.......... [OK] 51ms

REPL commands: .call <tool> <json>, .save <name>, .steps, .tools, .help, .exit. Recordings stored in ~/.mcpspec/recordings/.

mcpspec mock <recording>

Start a mock MCP server from a saved recording. Generate standalone .js files for portable, dependency-light mocks.

# Start mock server (stdio transport)
mcpspec mock my-api

# Use as a drop-in server replacement
mcpspec test --server "mcpspec mock my-api" ./tests.yaml

# Generate standalone .js file
mcpspec mock my-api --generate ./mock-server.js
node mock-server.js

# Options
mcpspec mock my-api --mode sequential          # Tape-style matching
mcpspec mock my-api --latency original         # Simulate original timing
mcpspec mock my-api --latency 100              # Fixed 100ms delay
mcpspec mock my-api --on-missing empty         # Return empty for unrecorded tools

Match mode: exact input match first, then next queued response per tool. Sequential mode: tape/cassette style — responses in recorded order. Generated files only require @modelcontextprotocol/sdk.

mcpspec docs <server>

Auto-generate documentation from server introspection. Lists all tools with input schemas and all resources with URIs.

mcpspec docs "npx my-server"                  # Markdown to stdout
mcpspec docs "npx my-server" --format html     # HTML output
mcpspec docs "npx my-server" --output ./docs   # Write to directory

Generated docs include: server name, version, description, all tools (name, description, JSON Schema input), and all resources (URI, name, description).

mcpspec compare / baseline

Save baselines and detect regressions between test runs. Shows regressions, fixes, new tests, and removed tests.

mcpspec baseline save main                 # Save current run
mcpspec baseline list                      # List baselines
mcpspec test --baseline main               # Compare against baseline
mcpspec compare --baseline main            # Explicit comparison
mcpspec compare <run-id-1> <run-id-2>      # Compare two runs

mcpspec init [directory]

Scaffold a new project with templates. Interactive wizard when no flags provided.

mcpspec init                               # Interactive wizard
mcpspec init ./my-project
mcpspec init --template minimal            # Minimal: single test
mcpspec init --template standard           # Standard: best practices
mcpspec init --template full               # Full: environments, tags, extraction

mcpspec ci-init

Generate CI pipeline configuration with an interactive wizard or flags. Auto-detects platform from project structure.

mcpspec ci-init                                # Interactive wizard
mcpspec ci-init --platform github              # GitHub Actions
mcpspec ci-init --platform gitlab              # GitLab CI
mcpspec ci-init --platform shell               # Shell script
mcpspec ci-init --checks test,audit,score      # Choose checks
mcpspec ci-init --server "npx my-server"       # Server for audit/score/bench
mcpspec ci-init --fail-on medium               # Audit severity gate
mcpspec ci-init --min-score 70                 # MCP Score threshold
mcpspec ci-init --force                        # Overwrite existing files

Auto-detects platform from .github/ or .gitlab-ci.yml. GitLab --force surgically replaces only the mcpspec job, preserving other jobs. Supports test, audit, score, and bench checks.

Collection Format

YAML-based test definitions — simple for quick tests, powerful when you need it.

Simple Format

name: My Tests
server: npx my-mcp-server

tests:
  - name: Basic call
    call: tool_name
    with:
      param: value
    expect:
      - exists: $.result
      - equals: [$.status, "ok"]

  - name: Handle errors
    call: tool_name
    with:
      bad_param: null
    expectError: true

Advanced Format

schemaVersion: "1.0"
name: Comprehensive Tests

server:
  command: npx
  args: ["my-mcp-server"]
  env:
    NODE_ENV: test

environments:
  dev:
    variables:
      BASE_PATH: /tmp/dev
  staging:
    variables:
      BASE_PATH: /tmp/staging

defaultEnvironment: dev

tests:
  - id: test-1
    name: Get data
    tags: [smoke, api]
    timeout: 5000
    retries: 2
    call: get_data
    with:
      path: "{{BASE_PATH}}/file.txt"
    assertions:
      - type: schema
      - type: exists
        path: $.content
      - type: latency
        maxMs: 1000
      - type: expression
        expr: "response.content.length > 0"
    extract:
      - name: fileContent
        path: $.content

Transport Configuration

MCPSpec supports 3 transport types for connecting to MCP servers.

stdio (default)

server:
  command: npx
  args: ["my-server"]
  env:
    NODE_ENV: test

Spawns child process, communicates via stdin/stdout

SSE

server:
  transport: sse
  url: http://localhost:3000/sse

HTTP Server-Sent Events for streaming

HTTP

server:
  transport: http
  url: http://localhost:3000/mcp

Streamable HTTP POST requests

Assertion Types

Type Description Example
schemaResponse is valid structuretype: schema
equalsExact match (deep comparison)path: $.id, value: 123
containsArray/string containspath: $.tags, value: "active"
existsPath exists and is not nullpath: $.name
matchesRegex matchpath: $.email, pattern: ".*@.*"
typeType checkpath: $.count, expected: number
lengthArray/string lengthpath: $.items, operator: gt, value: 0
latencyResponse time thresholdmaxMs: 1000
mimeTypeContent typeexpected: "image/png"
expressionSafe expression evalexpr: "response.total > 0"

Expressions use expr-eval — comparisons, logical operators, property access, math. No arbitrary code execution. Variables from extract are available in expressions.

Variables & Extraction

Environment Variables

environments:
  dev:
    variables:
      BASE_URL: http://localhost:3000
      API_KEY: dev-key-123

tests:
  - name: Use env variable
    call: fetch_data
    with:
      url: "{{BASE_URL}}/api/data"

Switch with --env staging

Variable Extraction

tests:
  - name: Create item
    call: create_item
    with: { name: "test" }
    extract:
      - name: newId
        path: $.id

  - name: Verify item
    call: get_item
    with:
      id: "{{newId}}"
    expect:
      - equals: [$.name, "test"]

Extract values from responses for use in later tests

CI/CD Integration

Run mcpspec ci-init to generate a pipeline, or use these examples directly.

GitHub Actions

Generate this with mcpspec ci-init --platform github

name: MCP Server Tests
on: [push, pull_request]

jobs:
  mcpspec:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'

      # Or add mcpspec as a devDependency
      # for version pinning
      - run: npm install -g mcpspec

      - name: Run tests
        run: mcpspec test --ci --reporter junit --output results.xml

      - name: Security audit
        run: mcpspec audit "npx my-server" --mode passive --fail-on high

      - name: MCP Score
        run: mcpspec score "npx my-server" --min-score 80

      - uses: mikepenz/action-junit-report@v4
        if: always()
        with:
          report_paths: results.xml

Architecture

TypeScript monorepo with 5 packages. pnpm workspaces + Turborepo.

@mcpspec/shared Types, Zod schemas, constants, exit codes
@mcpspec/core MCP client (3 transports), test runner, 10 assertions, 8 security rules, profiler, doc generator, scorer, recording/replay
@mcpspec/cli 13 CLI commands built with Commander.js + @inquirer/prompts for interactive wizards
@mcpspec/server Hono HTTP server with REST API + WebSocket for real-time updates, sql.js (WASM SQLite) storage
@mcpspec/ui React SPA with 10 pages — TanStack Router, TanStack Query, Tailwind CSS, shadcn/ui, dark mode
