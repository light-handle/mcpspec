Commands

10 commands for every stage of MCP server development.

mcpspec test [collection]

Run test collections with environments, tags, parallel execution, and multiple reporter formats.

mcpspec test                              # Uses ./mcpspec.yaml
mcpspec test ./tests.yaml                 # Specific file
mcpspec test --env staging                # Use staging variables
mcpspec test --tag @smoke                 # Filter by tag
mcpspec test --parallel 4                 # Parallel execution
mcpspec test --reporter junit --output results.xml
mcpspec test --baseline main              # Compare against baseline
mcpspec test --watch                      # Re-run on changes
mcpspec test --ci                         # CI mode

Reporters: console, json, junit, html, tap

mcpspec inspect <server>

mcpspec audit <server>

Security scanner with 6 vulnerability categories. Active modes require confirmation.

mcpspec audit "npx my-server"                       # Passive (safe)
mcpspec audit "npx my-server" --mode active          # Active probing
mcpspec audit "npx my-server" --mode aggressive      # Aggressive
mcpspec audit "npx my-server" --fail-on medium       # CI gate
mcpspec audit "npx my-server" --acknowledge-risk     # Skip prompt
mcpspec audit "npx my-server" --exclude-tools delete_user  # Skip tools
mcpspec audit "npx my-server" --dry-run              # Preview targets
RuleDetects
Path Traversal../../etc/passwd style attacks
Input ValidationMissing/malformed input handling
Resource ExhaustionCrash-inducing large payloads
Auth BypassAccess control circumvention
InjectionSQL/command injection in tool inputs
Info DisclosureLeaked paths, stack traces, secrets

mcpspec bench <server>

Performance benchmarking with statistical analysis.

mcpspec bench "npx my-server"                        # 100 iterations
mcpspec bench "npx my-server" --iterations 500
mcpspec bench "npx my-server" --tool read_file
mcpspec bench "npx my-server" --args '{"path":"/tmp/f"}'

Reports: min, max, mean, median, P95, P99, std dev, throughput (calls/sec)

mcpspec docs <server>

Auto-generate documentation from server introspection.

mcpspec docs "npx my-server"                  # Markdown to stdout
mcpspec docs "npx my-server" --format html     # HTML output
mcpspec docs "npx my-server" --output ./docs   # Write to directory

mcpspec compare / baseline

Save baselines and detect regressions between test runs.

mcpspec baseline save main                 # Save current run
mcpspec baseline list                      # List baselines
mcpspec compare --baseline main            # Compare against baseline
mcpspec compare <run-id-1> <run-id-2>      # Compare two runs

Scaffold a new project with templates.

Collection Format

YAML-based test definitions — simple for quick tests, powerful when you need it.

Simple Format

name: My Tests
server: npx my-mcp-server

tests:
  - name: Basic call
    call: tool_name
    with:
      param: value
    expect:
      - exists: $.result

  - name: Handle errors
    call: tool_name
    with:
      bad_param: null
    expectError: true

Advanced Format

schemaVersion: "1.0"
name: Comprehensive Tests

server:
  command: npx
  args: ["my-mcp-server"]
  env:
    NODE_ENV: test

environments:
  dev:
    variables:
      BASE_PATH: /tmp/dev

tests:
  - id: test-1
    name: Get data
    tags: [smoke, api]
    timeout: 5000
    retries: 2
    call: get_data
    with:
      path: "{{BASE_PATH}}/file.txt"
    assertions:
      - type: schema
      - type: exists
        path: $.content
      - type: latency
        maxMs: 1000
      - type: expression
        expr: "response.content.length > 0"
    extract:
      - name: fileContent
        path: $.content

Assertion Types

Type Description Example
schemaResponse is validtype: schema
equalsExact matchpath: $.id, value: 123
containsArray/string containspath: $.tags, value: "active"
existsPath existspath: $.name
matchesRegex matchpath: $.email, pattern: ".*@.*"
typeType checkpath: $.count, expected: number
lengthLength checkpath: $.items, operator: gt, value: 0
latencyResponse timemaxMs: 1000
mimeTypeContent typeexpected: "image/png"
expressionSafe expressionexpr: "response.total > 0"

Expressions use expr-eval — comparisons, logical operators, property access, math. No arbitrary code execution.

CI/CD Integration

Deterministic exit codes, multiple reporters, and ready-made workflow examples.

GitHub Actions

name: MCP Server Tests
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'

      - run: npm install -g mcpspec

      - name: Run tests
        run: mcpspec test --ci --reporter junit --output results.xml

      - name: Security audit
        run: mcpspec audit "npx my-server" --mode passive --fail-on high

      - uses: mikepenz/action-junit-report@v4
        if: always()
        with:
          report_paths: results.xml
